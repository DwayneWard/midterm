### Описание проекта

Тебе нужно разработать API для управления заказами (Order Management API) на FastAPI с использованием:
- Middleware для трассировки запросов и логирования
- Redis для кеширования данных
- Фоновых задач для асинхронной обработки
- Dependency Injection для управления зависимостями

API должно уметь:
- Создавать, читать, обновлять и удалять заказы (CRUD)
- Кешировать результаты запросов в Redis
- Использовать трассировку запросов через UUID
- Выполнять фоновые задачи
- Логировать все операции

### Структура проекта

Проект уже содержит базовую структуру и частично реализованный код. Твоя задача — завершить реализацию функций, помеченных как `TODO`, следуя указанным требованиям.

### Технические требования

- Python 3.12+
- Библиотеки: `fastapi`, `uvicorn`, `redis`, `apscheduler`, `pydantic`
- Использовать Redis для кеширования (локально или через Docker)
- Использовать Middleware для трассировки и логирования
- Использовать Dependency Injection через `Depends`
- Использовать фоновые задачи через `BackgroundTasks` и `APScheduler`

### Что нужно сделать

1. Изучи структуру проекта
2. Найди все `TODO` в коде
3. Реализуй недостающий функционал согласно описанию в комментариях
4. Убедись, что код работает корректно и обрабатывает ошибки
5. Запусти автотесты для проверки реализации (см. `TESTING.md`)
6. Сгенерируй отчет `TEST_REPORT.md` - это обязательный артефакт для сдачи

### Важные замечания

- Не изменяй существующий код без необходимости
- Следуй структуре проекта
- Обращай внимание на обработку ошибок и HTTP-статусы
- Код должен быть читаемым и структурированным
- Используй асинхронные операции где это возможно
