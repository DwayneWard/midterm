# Инструкция по запуску тестов

## Установка зависимостей

Перед запуском тестов необходимо установить все зависимости:

```bash
pip install -r requirements.txt
```

## Настройка Redis

Для работы тестов требуется Redis. Есть два варианта:

### Вариант 1: Локальный Redis

Убедитесь, что Redis запущен локально на порту 6379:

```bash
# Проверка, что Redis запущен
redis-cli ping
```

### Вариант 2: Redis через Docker

```bash
docker run -d -p 6379:6379 redis:latest
```

### Вариант 3: Использование fakeredis (для тестов)

Тесты используют `fakeredis` для мокирования Redis, поэтому реальный Redis не обязателен для запуска тестов.

## Запуск тестов

### Linux/Mac

```bash
./run_tests.sh
```

### Windows

```bash
run_tests.bat
```

### Вручную

```bash
# Запуск тестов с генерацией JSON отчета
pytest --json-report --json-report-file=test_results.json -v

# Генерация Markdown отчета
python tests/generate_report.py
```

## Результаты тестирования

После выполнения тестов будет создан файл **`TEST_REPORT.md`** с детальным отчетом по всем образовательным результатам.

Отчет включает:
- Общую статистику по тестам
- Результаты по каждому образовательному результату (ОР)
- Начисленные баллы за каждый ОР
- Итоговую оценку

## Структура отчета

Отчет содержит следующие разделы:

1. **Общая информация** - статистика по тестам
2. **Результаты по образовательным результатам** - детализация по каждому ОР
3. **Итоговая оценка** - общий балл и оценка

## Важные замечания

- Убедитесь, что все TODO в коде реализованы перед запуском тестов
- Проверьте, что Redis доступен (если используется реальный Redis)
- Файл `TEST_REPORT.md` является обязательным артефактом для сдачи работы
