# Практическая часть контрольной работы. Вариант 6
## Разработка на Python. Красный трек

### Общая информация

**Время выполнения:** 100 минут  
Практическая реализация проекта

### Описание проекта

Тебе нужно разработать веб-приложение для управления рецептами на FastAPI. Приложение должно уметь:
- Отображать список рецептов на главной странице
- Добавлять новые рецепты через форму
- Просматривать детальную информацию о рецепте
- Редактировать информацию о рецепте
- Удалять рецепты
- Обрабатывать ошибки и возвращать корректные HTTP-статусы

### Структура проекта

Проект уже содержит базовую структуру и частично реализованный код. Твоя задача — завершить реализацию функций, помеченных как `TODO`, следуя указанным требованиям.

### Технические требования

- Python 3.12+
- Библиотеки: `fastapi`, `uvicorn`, `jinja2`, `python-multipart`
- Использовать Pydantic для валидации данных
- Использовать Jinja2 для рендеринга HTML-шаблонов
- Хранение данных в памяти (без базы данных)

### Что нужно сделать

1. Изучи структуру проекта
2. Найди все `TODO` в коде
3. Реализуй недостающий функционал согласно описанию в комментариях
4. Убедись, что код работает корректно и обрабатывает ошибки
5. Запусти автотесты для проверки реализации (см. `TESTING.md`)
6. Сгенерируй отчет `TEST_REPORT.md` - это обязательный артефакт для сдачи

### Важные замечания

- Не изменяй существующий код без необходимости
- Следуй структуре проекта
- Обращай внимание на обработку ошибок и HTTP-статусы
- Код должен быть читаемым и структурированным

### Проверка выполнения

После реализации всех TODO запусти автотесты:

```bash
# Установи зависимости
pip install -r requirements.txt

# Запусти тесты и сгенерируй отчет
./run_tests.sh  # Linux/Mac
# или
run_tests.bat  # Windows
```

Будет создан файл **`TEST_REPORT.md`** с результатами проверки всех образовательных результатов. Этот файл является обязательным артефактом для сдачи контрольной работы.

Подробная инструкция по тестированию находится в файле `TESTING.md`.
