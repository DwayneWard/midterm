# Практическая часть контрольной работы. Вариант 3
## Разработка на Python. Основной

### Общая информация

**Время выполнения:** 100 минут  
**Тип работы:** Практическая реализация проекта

### Описание проекта

Вам нужно доработать Telegram-бота для получения новостей по категориям. Бот должен уметь:
- Получать новости по категории через внешний API
- Сохранять избранные категории новостей пользователя
- Показывать список сохранённых категорий с помощью интерактивных кнопок
- Обрабатывать ошибки при работе с API

### Структура проекта

Проект уже содержит базовую структуру и частично реализованный код. Ваша задача — завершить реализацию функций, помеченных как `TODO`, следуя указанным требованиям.

### Технические требования

- Python 3.12+
- Библиотеки: `pyTelegramBotAPI` (telebot), `requests`
- Использовать внешний API для получения новостей (NewsAPI - бесплатный план доступен, требует API ключ)
- Бот должен быть реализован синхронно (без использования async/await)

### Что нужно сделать

1. Изучите структуру проекта
2. Найдите все `TODO` в коде
3. Реализуйте недостающий функционал согласно описанию в комментариях
4. Убедитесь, что код работает корректно и обрабатывает ошибки
5. Запустите автотесты для проверки реализации (см. `TESTING.md`)
6. Сгенерируйте отчет `TEST_REPORT.md` - это обязательный артефакт для сдачи

### Важные замечания

- Не изменяйте существующий код без необходимости
- Следуйте структуре проекта
- Обращайте внимание на обработку ошибок
- Код должен быть читаемым и структурированным

### Проверка выполнения

После реализации всех TODO запустите автотесты:

```bash
# Установите зависимости
pip install -r requirements.txt

# Запустите тесты и сгенерируйте отчет
./run_tests.sh  # Linux/Mac
# или
run_tests.bat  # Windows
```

Будет создан файл **`TEST_REPORT.md`** с результатами проверки всех образовательных результатов. Этот файл является обязательным артефактом для сдачи контрольной работы.

Подробная инструкция по тестированию находится в файле `TESTING.md`.
